T5A4. Reach V0.1

Work as a team to deliver V0.1. Note the following:
  • Refer to our instructions in the handbook to find possible workflow options.
  • Divide components (GUI, Model, etc.) among team members. Learn the code of your component. In V0.1 (and V0.2), try to limit your work to your component only.
  • Change code in small steps, run tests after each step, commit each step, and merge often.
  • Start using the "Author Tagging" mechanism mentioned in V0.4 Requirement . As your team is likely to generate a large amount of code starting, you are strongly encouraged to take some time to tag your code as described. It may be very cumbersome and tedious to do this afterwards.
Submission:
As your project starts to diverge from AddressBook codebase, it is no longer meaningful to use PRs to indicate progress.
Instead, push all the relevant changes to the main branch of your repo once it reaches V0.1 requirements. Remember to use git tag to annotate that commit as V0.1.
