Pre-Tutorial Learning Activities

T7A1. Use project schedule tracking

If you haven't done so yet, create labels and milestones in your team repo as given in our issue tracker setup instructions.

Start using GitHub for managing your project schedule. Here is a follow-up guide for you. Tips: Set your internal milestones to be earlier than the formal schedule to give your team some buffers.

Note that you'll lose marks if you don't 'reach' the weekly milestone starting from V0.3.

T7A2. Check the contributions

You can check each person's contribution using the Github graphs. Note that we do not use number of commits shown here for grading. Instead, we use a different mechanism to extract code written by each student.

T7A3. Reach V0.3

Work as a team to deliver V0.3. Note the following:
  • Update both the developer guide (especially the appendices) and the user guide to match your product development.
  • As most of you are about to improve the design of your product at this point, we have compiled a number of product design tips for you to consider.
  • Use the "Author Tagging" mechanism mentioned in V0.4 Requirement . As your team is likely to generate a large amount of code starting from V0.3, you are strongly encouraged to take some time to tag your code as described. It may be very cumbersome and tedious to do this afterwards.
