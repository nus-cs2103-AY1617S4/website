T3A4. Set up the sample project

Ensure your team's GitHub organization name is as specified in our requirements. Follow the repo setup and issue tracker setup instructions in our handbook to set up the repo for your project.
  1. Ensure you can pass all the tests locally.
  2. Set up Travis for your fork.
  3. Read the 'Design' Section of the Developer Guide to get a high-level understanding of the design before you start reading the code.

T3A5. Get started on V0.0

Prepare V0.0 Deliverables

Suggested sequence : document product survey → brainstorm user stories → prioritize use stories → select user stories to include in your product → design the product → write the user guide
Fill up Glossary and NFRs in parallel to the above.

When creating these documents, initially you may use a simple collaboration medium such as Google Docs. You can transfer the content to your GitHub repo after the content has become somewhat stable.

Tip: Find an Eclipse plugin to preview Markdown documents.
